## Overview

This project involved building a communication app for cruise ship personnel and guests. The focus was on real-time interactions and a reliable offline-first architecture.

## Key Features

- **Built from Scratch**: Developed a Flutter app with 10+ screens for seamless guest-crew communication.
- **Real-Time Communication**: Integrated a WebSocket service for instant data flow.
- **Offline Data Management**: Implemented Hive for local storage, ensuring accessibility even without network connectivity.

## Technologies Used

- **Flutter & Dart**: For mobile development.
- **BLoC Architecture**: For structured state management.
- **WebSocket**: For real-time data exchange.
- **Hive**: For lightweight local database storage.

## Challenges and Learnings

Ensuring real-time communication while maintaining reliability in fluctuating network conditions was a key challenge.

## Outcome

The developed app significantly improved communication efficiency on cruise ships, reducing delays and enhancing guest experience.
